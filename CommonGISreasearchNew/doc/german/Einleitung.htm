Beschreibung:

Descartes, ein Java basiertes und im Web nutzbares GIS (Geographisches Informations System), das in besonderer Weise die Erstellung und interaktive Analyse von thematischen Karten unterstützt. Aus einer Menge raumbezogener statistischer Größen braucht sich der Benutzer nur die ihn interessierenden auswählen, und Descartes erstellt unter Anwendung einer kartographischen Wissensbasis automatisch eine Menge von Karten, in der die Größen und ihre Beziehungen visuell erfaßt werden können.

Es ist ein umgebungsunabhängiges Programm. Seine umgebungsspezifischen Installationen heißen Applikationen. Vom System aus gesehen ist eine Applikation eine Reihe von Informationsdateien, die zusammen mit Bedienungsdateien, welche dem System mitteilen wie die Daten handzuhaben sind, visualisiert und analysiert werden.
Eine Applikation sollte zwei Arten von Informationsdateien erhalten:
&middot; Tabellen mit Attributdaten welche räumlichen Objekten (Bezirke, Städte, Regionen, etc.) zugeordnet sind. Die Objekte werden über ihre Bezeichner aus Tabellen heraus benannt. Eine Tabelle sollte eine Zeile mit Objektbezeichner haben.
&middot; geographische Schichten, d.h. Dateien, welche Punkte, Linien oder Flächen (Polygone) in Vektorformat enthalten. Jeder Tabelle sollte eine geographische Schicht zugeordnet werden, welche die räumlichen Objekte dieser enthält. Diese Objekte sollten in der Tabelle und in der geographischen Schicht die selben Bezeichner haben.
Eine Applikation kann einige Tabellen enthalten, die der selben geographischen Schicht zugeordnet sind, als auch unterschiedlichen Schichten. Jedoch kann eine Tabelle nicht auf mehrere Schichten zugleich zugeordnet sein. In einer Applikation können beliebig viele Schichten vorkommen. Die Schichten, die keine Objekte einer Tabelle enthalten, könnten als geographischer Hintergrund der Präsentation dienen. Welche Schichten in einer Applikation vorhanden sind, wird in der Legende der Karte dargestellt.

Descartes ermöglicht es mehrere Darstellung mit unterschiedlichen Attributen der selben Informationsmenge, oder gar des selben Attributes in unterschiedlicher Darstellungsweise zu erzeugen. Demnach kann der Benutzer eine oder mehrere Karten und verschiedene nicht graphische Darstellungen wie dot plots (Punktgraphen), scatter plots (Streugraphen), oder parallel coordinate plots (Graphen mit parallelen Koordinaten) gleichzeitig geöffnet haben. Alle diese Darstellungen sind dynamisch verbunden. D.h. wenn der Benutzer mit der Maus auf ein Objekt in einen der Darstellungen zeigt, werden alle graphischen Elemente des zugehörigen Objektes in sämtlichen Darstellungen (Konturen in der Karte, Punkte in den Graphen, Linien in dem Graphen mit parallelen Koordinaten) simultan hervorgehoben. Mehr Informationen zu dynamische Verknüpfungen kann man hier finden.
